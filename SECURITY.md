# Security Policy

## Supported Versions

Only the latest minor release on `main` receives security fixes. Pinned or
forked installs are the user's responsibility.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security reports.

Email: **tjdhkdrb@gmail.com** with the subject line
`[harness-loom] security report`. Include:

- a description of the issue and its impact
- steps to reproduce (minimal repro preferred)
- the commit hash or release tag you tested against

You can expect an acknowledgement within 7 days. If the report is confirmed,
we will coordinate a fix and disclosure timeline with you before publishing
any details.

## Scope

In scope:

- factory scripts under `skills/*/scripts/`
- runtime templates installed into target repositories
- plugin manifests (`.claude-plugin/`, `.codex-plugin/`, `.agents/plugins/`)

Out of scope:

- third-party model/platform providers (Claude Code, Codex CLI, Gemini CLI)
- user-authored producer/reviewer pairs generated by `/harness-pair-dev`
- vulnerabilities that require a pre-compromised local machine
